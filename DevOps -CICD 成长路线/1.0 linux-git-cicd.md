##  第 1 周：Linux 生存技能（Day 1–7）

### 必会命令（每天敲）

```bash
ls  cd  pwd
cp  mv  rm
grep  awk  sed
tail -f
ps  top
netstat / ss
```

### Day 1–3

* 文件操作
* 管道 `|`
* 重定向 `> >>`

### Day 4–5

* 进程 / 端口
* systemctl
* 日志定位

### Day 6–7（关键）

* 权限：

  ```bash
  chmod  chown
  ```
* 用户 / sudo

 目标：

> 给你一个服务端口不通，你能找到原因

---

##  第 2 周：Git 实战（Day 8–14）

### 必会命令

```bash
git clone
git status
git add .
git commit
git pull --rebase
git tag
git log --oneline
```

### 必做实战

* 本地建仓库
* 至少制造一次冲突
* 自己解决冲突

### 一定要理解的 3 件事

* commit ≠ push
* branch ≠ copy
* rebase ≠ merge

 目标：

> 你敢在生产仓库里动代码

---

##  第 3 周：CI/CD 概念 + YAML（Day 15–21）

### 必懂概念

* Pipeline
* Stage
* Job
* Artifact
* Runner / Agent

### YAML 核心

```yaml
stages:
  - test
  - build

job-test:
  stage: test
  script:
    - echo "test"
```

### 作业

* 看懂一份 GitLab CI / GitHub Actions
* 能解释每一行在干嘛

 目标：

> 不再害怕 pipeline 文件

---

##  第 4 周：第一条流水线（Day 22–30）

### 实战目标

> **git push → pipeline 自动跑**

### 示例（最小可用）

```yaml
stages:
  - test

test-job:
  stage: test
  script:
    - echo "hello ci"
```

### 你要做到

* pipeline 自动触发
* 失败能看日志
* 知道为什么失败

 终极目标：

> 你已经是“会用 CI 的人”
