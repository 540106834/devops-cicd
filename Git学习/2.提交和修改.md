# 第一次摸 Git 的说明书

# 一、先搞清楚 3 个地方（最重要）

在 Git 里，你的文件**同时可能存在在 3 个“地方”**：

```
工作区        暂存区        仓库
(你正在改) → (准备提交) → (已经存档)
```

一句人话版：

- **工作区**：你正在用编辑器改文件
- **暂存区**：你告诉 Git「这次改动我确认要保存」
- **仓库**：Git 已经帮你存好了一个历史版本

 **修改 = 工作区变化**
 **提交 = 暂存区 → 仓库**

------

# 二、从 0 创建仓库

```bash
mkdir git-demo
cd git-demo
git init

这一步干了什么？
- 创建了 `.git/` 目录
- Git 开始“记账”
```

------

# 三、第 1 次修改（什么都还没提交）

新建文件：

```bash
echo "hello git" > note.txt
```

现在问 Git：

```bash
git status
```

你会看到：

```text
Untracked files:
  note.txt
```

解释：

- **Untracked** = Git 还不知道这个文件存在
- 此时：文件只在 **工作区**

------

# 四、把修改“告诉”Git（add）

```bash
git add note.txt
```

再看状态：

```bash
git status
```

结果：

```text
new file: note.txt
```

发生了什么？

- 文件被放进了 **暂存区**
- 但还没真正保存成历史

📌 类比：

> `git add` = 把文件放进“待保存清单”

------

# 五、真正保存一次（commit）

```bash
git commit -m "第一次提交 note.txt"
```

这一步：

- Git 创建了一个**版本**
- 永久记录当前内容

现在结构是：

```
仓库里：
note.txt = "hello git"
```

------

# 六、第 2 次修改（最常见场景）

改文件：

```bash
echo "learning git" >> note.txt
```

文件内容变成：

```text
hello git
learning git
```

此时状态：

```bash
git status
```

显示：

```text
modified: note.txt
```

解释：

- 文件改了
- 但 Git 还没被通知

📌 这就是 **修改 ≠ 提交**

------

# 七、查看你到底改了什么（diff）

```bash
git diff
```

你会看到：

```diff
+learning git
```

👉 这是**工作区 vs 上一次提交**的差异

------

# 八、再次 add（确认这次修改要提交）

```bash
git add note.txt
```

此刻状态：

```bash
git status
```

显示：

```text
Changes to be committed:
  modified: note.txt
```

说明：

- 修改进入了暂存区
- 等你拍板

------

# 九、再次 commit（第二个版本）

```bash
git commit -m "添加学习 git 内容"
```

现在 Git 里有 **两个历史版本**：

```
版本 1: hello git
版本 2: hello git
        learning git
```

📌 Git **不会覆盖旧版本**
📌 它是「新增一个历史节点」

------

# 十、反复修改 + 提交 = 正常流程

你以后每天都在重复这套动作：

```
改文件
↓
git status
↓
git diff
↓
git add
↓
git commit
```

就像：

> 写作 → 校对 → 装订 → 存档

------

# 十一、初学者最容易混的 3 个问题

### ❓1. 我改了文件，为什么 git commit 没反应？

👉 因为你**没 git add**

------

### ❓2. add 之后还能继续改吗？

👉 可以

- 新改的内容 **不在暂存区**
- 需要再次 `git add`

------

### ❓3. commit 写错了会怎样？

👉 代码没丢，只是说明不好看
后面可以改（进阶再讲）

------

# 十二、记住这三句就够了

1️⃣ **修改只发生在工作区**
2️⃣ **add 决定哪些修改要提交**
3️⃣ **commit 才是真正存历史**

**`git diff` 三种对比:**

| 命令                | 比较的两个区域  | 白话解释                         |
| ------------------- | --------------- | -------------------------------- |
| `git diff`          | 工作区 ↔ 暂存区 | 我现在改了什么，还没准备提交     |
| `git diff --staged` | 暂存区 ↔ 仓库   | 我准备提交的内容，和上次提交差哪 |
| `git diff HEAD`     | 工作区 ↔ 仓库   | 从上次提交到现在，一共改了啥     |
