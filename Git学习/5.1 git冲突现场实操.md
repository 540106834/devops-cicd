
## 一句话结论先给你

> **`git branch` 不显示任何分支，通常只有一个原因：
> 你还没有任何提交（commit）。**

---

## 现场 1：刚 `git init`，立刻 `git branch`

### 复现

```bash
git init
git branch
```

### 结果

```text
# 什么都没有
```

### 原因

* Git 的分支本质是 **指向某个 commit 的指针**
* 你现在 **一个 commit 都没有**
* 没有“可指向的地方”，分支就不存在

📌 这是**最常见原因**。

---

## 解决办法（立刻见效）

```bash
echo "init" > README.md
git add .
git commit -m "first commit"
```

再来一次：

```bash
git branch
```

你会看到：

```text
* main
```

牌子挂出来了 

---

## 现场 2：你在“游离 HEAD”（detached HEAD）

### 判断

```bash
git status
```

如果你看到：

```text
HEAD detached at a1b2c3d
```

### 这时：

```bash
git branch
```

可能只显示已有分支，**但当前不在任何分支上**

### 回到分支

```bash
git checkout main
```

---

## 现场 3：你以为有分支，其实是远程分支

```bash
git branch
```

只看本地分支

```bash
git branch -a
```

看全部（本地 + 远程）

```bash
git branch -r
```

只看远程

📌 很多人卡在这一步。

---

## 现场 4：仓库根本不是 Git 仓库

```bash
git branch
```

如果看到：

```text
fatal: not a git repository
```

检查：

```bash
ls -a
```

有没有 `.git` 目录？

---

## 最快自检流程（照着敲）

```bash
git status
git log --oneline
git branch -a
```

* `log` 没内容 → 没 commit
* `branch -a` 有但 `branch` 没 → 只有远程分支
* `detached HEAD` → 先 checkout 回来
